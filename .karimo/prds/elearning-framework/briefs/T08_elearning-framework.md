## Objective

## Context

**Denormalized filter fields are already on the documents.** `gender`, `ageGroup`, `healthWorkerType`, `country` are written onto `lessonProgress` and `surveyResponse` documents at creation time by T03, T05, and T06 respectively. Do NOT query the Clerk API for filter dimensions at dashboard query time — use these document-level fields directly in GROQ filter params.

**Role scoping:**
- `program_manager`: ALL queries must include `&& country == $userCountry` as an additional GROQ filter. The `$userCountry` param comes from `sessionClaims.metadata.country`.
- `partner_donor`: read-only but unrestricted by country (same as admin for data scope in PRD-A — confirm with stakeholder if this assumption changes).
- `system_admin`: queries unrestricted by country.
- `learner`: return 403 immediately — no dashboard data access.

**NPS promoter definition:** `npsScore >= 9` (NOT `csatScore`). NPS formula: `(count of promoters / total responses) * 100`. A promoter is a survey respondent with `npsScore >= 9`.

**`ageGroup` derivation:** The `ageGroup` field stored on `lessonProgress` / `surveyResponse` uses these bands (derived from DOB at write time by the progress/commit/survey Route Handlers): `'<18'`, `'18-24'`, `'25-34'`, `'35-44'`, `'45+'`. T08 queries filter against these string values.

**Sanity CDN client for reads:** Use `client` (not `writeClient`) for all dashboard GROQ queries.

**Typed queries:** Import types from `sanity.types.ts` (generated by T01). Use Sanity's typed GROQ result patterns. The `DashboardMetrics` interface in `src/types/lms.ts` (from T01) defines the return shape.

**GROQ aggregation limitations:** Sanity GROQ does not have a native `COUNT(DISTINCT)` or date-truncation function like SQL. For DAU and "new users by country" metrics, use array operations:
- DAU: `count(*[_type == "lessonProgress" && completedAt > $yesterday].userId)` — note this counts progress documents, not distinct users; for distinct users use `array::unique(...)` if available in your Sanity version, or accept the approximation.
- New users by country: aggregate `lms_registered` is tracked via Clerk, not Sanity. For this metric, count `lessonProgress` documents created in the last month grouped by `country`, using GROQ's `dateTime` functions.

**Query file structure:** Export individual named query functions from `dashboardQueries.ts`. Each function accepts `DashboardFilters & { userRole: LmsRole; userCountry: string }` and returns a typed GROQ result. This allows T09 to call individual metrics independently (for streaming/parallel loading) rather than requiring all 8 in one call.

## Implementation Steps

### Step 1 — Create `src/sanity/lib/dashboardQueries.ts`

Import `client` from `src/sanity/lib/client.ts` and `groq` from `groq` (or from `next-sanity`).

Define a helper to apply role scoping to a base filter string:
```typescript
function applyRoleScope(baseFilter: string, userRole: LmsRole, userCountry: string): string {
  if (userRole === 'program_manager') {
    return `${baseFilter} && country == "${userCountry}"`
  }
  return baseFilter
}
```

Define and export the following query functions. Each accepts `filters: DashboardFilters & { userRole: LmsRole; userCountry: string }`:

**1. `getCsatAvg(filters)`**
```groq
avg(*[_type == "surveyResponse" {roleScope} {filterClauses}].csatScore)
```
Returns: `number`

**2. `getNpsScore(filters)`**
```groq
// Promoters: npsScore >= 9
// Result: (promoters / total) * 100
count(*[_type == "surveyResponse" {roleScope} {filterClauses} && npsScore >= 9]) * 100 / count(*[_type == "surveyResponse" {roleScope} {filterClauses}])
```
Returns: `number` (percentage)

**3. `getKnowledgeGain(filters)`**
```groq
avg(*[_type == "lessonProgress" {roleScope} {filterClauses} && defined(postTestScore) && defined(preTestScore)][].postTestScore - preTestScore)
```
Returns: `number` (average score delta)

**4. `getDau(filters)`**
```groq
count(*[_type == "lessonProgress" {roleScope} {filterClauses} && completedAt > $yesterday])
```
Params: `{ yesterday: new Date(Date.now() - 86400000).toISOString() }`
Returns: `number`

**5. `getConversionRate(filters)`**
```groq
// Enrolled = has any lessonProgress for course
// Completed = has lessonProgress with completed == true for final lesson
// Approximation: count(completed lessonProgress) / count(total lessonProgress) * 100
count(*[_type == "lessonProgress" {roleScope} {filterClauses} && completed == true]) * 100 / count(*[_type == "lessonProgress" {roleScope} {filterClauses}])
```
Returns: `number` (percentage)

**6. `getRetentionRate(filters)`**
```groq
count(*[_type == "lessonProgress" {roleScope} {filterClauses} && completedAt > $twelveMonthsAgo])
```
Params: `{ twelveMonthsAgo: new Date(Date.now() - 365 * 86400000).toISOString() }`
Returns: `number` (count of active users in last 12 months — approximation)

**7. `getNewUsersByCountry(filters)`**
```groq
*[_type == "lessonProgress" {roleScope} && defined(country)] {
  country,
  "month": string::slice(completedAt, 0, 7)
} | order(month desc)
```
Post-process in JavaScript to group by country and month.
Returns: `Array<{ country: string; month: string; count: number }>`

**8. `getKnowledgeBaseGrowth(filters)`**
```groq
count(*[_type == "course"])
```
Returns: `number` (total course count — month-over-month growth requires created date tracking; return total count for now)

### Step 2 — Handle filter clauses

For each query, build optional filter clauses from `DashboardFilters`:
```typescript
const filterClauses = [
  filters.gender ? `&& gender == "${filters.gender}"` : '',
  filters.ageGroup ? `&& ageGroup == "${filters.ageGroup}"` : '',
  filters.learnerType ? `&& healthWorkerType == "${filters.learnerType}"` : '',
  // country filter only added if not already applied by role scope
  filters.country && filters.userRole !== 'program_manager'
    ? `&& country == "${filters.country}"` : '',
].filter(Boolean).join(' ')
```

Note: GROQ injection risk — these filter values come from the user-facing dashboard. Do NOT interpolate `filters.*` values directly into GROQ strings. Use GROQ params (`$param`) for all filter values:
```groq
*[_type == "lessonProgress" && gender == $gender]
```
Pass params as the second argument to `client.fetch()`.

### Step 3 — Create `src/app/api/lms/dashboard/route.ts`

1. Auth guard: return 401 if no `userId`.
2. Read role and country from `sessionClaims`: `const { role, country } = sessionClaims?.metadata ?? {}`.
3. Return 403 if `role === 'learner'` or `role === undefined`.
4. Parse URL search params for `DashboardFilters`: `country`, `gender`, `ageGroup`, `learnerType`.
5. For `program_manager`: override `country` filter with `userCountry` regardless of search params (country is pre-filled and disabled in T09 UI, but enforce server-side too).
6. Call all query functions in parallel:
   ```typescript
   const [csatAvg, npsScore, knowledgeGain, dau, conversionRate, retentionRate, newUsersByCountry, knowledgeBaseGrowth] = await Promise.all([
     getCsatAvg(filtersWithRole),
     getNpsScore(filtersWithRole),
     getKnowledgeGain(filtersWithRole),
     getDau(filtersWithRole),
     getConversionRate(filtersWithRole),
     getRetentionRate(filtersWithRole),
     getNewUsersByCountry(filtersWithRole),
     getKnowledgeBaseGrowth(filtersWithRole),
   ])
   ```
7. Return `200` with `DashboardMetrics` JSON object.
